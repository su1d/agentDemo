# 多智能体协调系统设计方案

## 概述

基于现有架构（Java 21 Spring Boot + Python LangGraph + Vue 3）扩展为多智能体系统。
同质 Agent 分工模式 + Java 后端统一调度 + **7 种协作模式**。

## 架构

```
用户 (Vue 3, port 5173)
    ↓ HTTP
Java 调度器 (Spring Boot, port 8080) ←→ Orchestrator Worker (Python)
    │                                    ├── Agent-A: 计算/数学 (calculator)
    │                                    ├── Agent-B: 天气/地理 (weather)
    │                                    ├── Agent-C: 总结/输出 (summarizer)
    │                                    ├── Agent-D: 搜索 (searcher)
    │                                    └── Agent-E: 通用对话 (assistant)
    │
    ├── POST /api/chat                    ← 统一入口，通过 action 参数切换模式
    ├── GET  /api/agents                  ← 查看所有 Agent 状态
    ├── GET  /api/info                    ← 系统信息（含可用模式列表）
    └── GET  /api/health                  ← 健康检查
```

## 7 种协作模式

| 模式 | Action | 说明 | 适用场景 |
|------|--------|------|----------|
| 🎯 自动路由 | `chat` | LLM/关键词判断最合适的单一 Agent | 简单问答 |
| 🔗 链式编排 | `orchestrate` | 多个 Agent 依次处理，前一个输出作为下一个输入 | 多步骤任务（搜索→计算→总结） |
| ⚡ 并行汇总 | `parallel` | 多个 Agent 同时处理 → Summarizer 汇总 | 多角度分析 |
| 🏛️ 辩论 | `debate` | 多 Agent 多轮辩论 → Summarizer 出结论 | 复杂决策 |
| 🔍 批评链 | `critique_chain` | A 生成 → B 批评 → A 改进（多轮） | 质量迭代 |
| 📊 投票共识 | `voting_consensus` | 各出方案 → Summarizer 评估选最佳 | 方案选择 |
| 💡 头脑风暴 | `brainstorm` | 各自由创意 → Summarizer 归类整理 | 创意发散 |

## 通信协议

每个 Worker 进程使用 stdin/stdout JSON 协议。
Java 端通过 `AgentWorkerManager` 管理所有 Worker 的生命周期。

### 请求格式

```json
{"message": "问题", "action": "debate", "roles": ["searcher", "calculator"], "rounds": 2}
```

### 回复格式（协作模式返回结构化 JSON）

```json
{"type": "reply", "content": "{ \"mode\": \"parallel\", \"individual_results\": {...}, \"summary\": \"...\" }"}
```

## 数据处理

### agent_execution_log 表

每次 Agent 执行记录 input_text, output_text, status, duration_ms，用于：
- 追踪协作链中的每个环节
- 性能监控和调优
- 失败重试的依据

### agent_workflow 表

持久化工作流定义，支持：
- 预设协作模式配置
- 节点排序和超时设置
- 输入/输出映射模板

## 重试与容错

- 每个 Agent 调用最多重试 2 次
- 超时时间 120s（复杂协作可能耗时较长）
- 超时后自动重启 Worker 进程再重试
- 前端运行时检测后端状态（10s 轮询 /api/info）
